# NepForge 项目概述

## 1. 项目名称

**NepForge**

## 2. 项目类型

NepForge 是一个基于 **Spring Boot + Vue 3** 的前后端分离装机与数码产品社区平台，面向对电脑硬件、装机方案、数码产品参数、硬件文章和装机交流感兴趣的用户。

项目不仅关注传统台式电脑配件，也会逐步扩展到显示器、键盘、鼠标、外设、笔记本、数码产品等更广泛的消费电子产品信息。

## 3. 项目背景

在电脑硬件与数码产品领域，用户通常会面临以下问题：

1. 配件信息分散，参数对比不方便。
2. 装机配置需要手动记录，缺少统一管理工具。
3. 新手用户不了解配件之间的搭配关系。
4. 优质装机经验、硬件评测、选购建议分散在不同平台。
5. 用户希望保存自己感兴趣的文章、配件和装机单。
6. 学生开发者需要一个贴近企业真实开发流程的完整项目，用于学习、实践和求职展示。

NepForge 的目标是将“配件库、装机单、参数对比、专栏内容、社区交流、用户收藏”整合到一个完整平台中，并通过真实的工程规范模拟企业级项目开发流程。

## 4. 项目目标

### 4.1 业务目标

NepForge 计划实现以下业务能力：

1. 展示硬件与数码产品信息。
2. 支持用户浏览、筛选、搜索配件。
3. 支持配件参数详情展示。
4. 支持用户创建、编辑、保存自己的装机单。
5. 支持装机单自动统计总价和总功耗。
6. 支持文章发布、装机指南、硬件评测、社区讨论。
7. 支持文章评论和评论回复。
8. 支持点赞、收藏、收藏夹。
9. 支持用户个人主页。
10. 支持管理员维护配件、分类、文章、评论、用户等数据。
11. 为后续交互式装机教学模块预留扩展空间。

### 4.2 技术目标

项目开发不仅要实现功能，还要模拟企业级开发流程：

1. 使用 GitHub 管理代码。
2. 使用规范化分支模型进行开发。
3. 编写完整项目文档。
4. 编写数据库设计文档。
5. 编写接口文档。
6. 使用统一响应结构和全局异常处理。
7. 使用 JWT 完成登录认证。
8. 使用 RBAC 完成权限控制。
9. 使用 Redis 优化热点数据。
10. 使用 Docker Compose 模拟真实部署环境。
11. 使用 GitHub Actions 尝试自动化构建。
12. 形成可用于实习面试展示的完整项目。

## 5. 项目用户角色

### 5.1 游客

游客是未登录用户，可以进行基础浏览。

可使用功能：

1. 浏览首页。
2. 浏览公开配件信息。
3. 查看公开装机单。
4. 查看已发布文章。
5. 查看评论内容。

不可使用功能：

1. 点赞。
2. 收藏。
3. 创建装机单。
4. 发布文章。
5. 发表评论。
6. 进入个人中心。

### 5.2 普通用户

普通用户是注册并登录后的核心使用者。

可使用功能：

1. 修改个人资料。
2. 创建、编辑、删除自己的装机单。
3. 收藏文章、配件、装机单。
4. 管理自己的收藏夹。
5. 点赞文章、配件、装机单和评论。
6. 发布文章。
7. 评论文章。
8. 回复评论。
9. 访问个人主页。
10. 查看自己的点赞记录和收藏记录。

### 5.3 版主

版主负责维护社区秩序。

可使用功能：

1. 拥有普通用户权限。
2. 删除违规评论。
3. 下架违规文章。
4. 处理用户举报。
5. 管理部分社区内容。

### 5.4 管理员

管理员拥有最高系统管理权限。

可使用功能：

1. 管理用户。
2. 管理角色。
3. 管理配件分类。
4. 管理配件信息。
5. 管理文章。
6. 管理评论。

8. 查看系统数据统计。

10. 管理系统配置。

## 6.2 配件库模块

该模块负责硬件与数码产品数据展示。

1. 配件分类树。
2. 配件列表。
3. 配件详情。

## 7. 项目技术架构

## 7.1 整体架构

项目采用前后端分离架构：

```text
用户浏览器
   ↓
Vue 3 前端应用
   ↓ HTTP / HTTPS
Spring Boot 后端服务
   ↓
MySQL / Redis / 对象存储
```

## 7.2 后端架构

推荐后端技术栈：

| 类型     | 技术                        |
| -------- | --------------------------- |
| 基础框架 | Spring Boot                 |
| 开发语言 | Java                        |
| ORM      | MyBatis-Plus 或 MyBatis     |
| 数据库   | MySQL 8.x                   |
| 缓存     | Redis                       |
| 权限认证 | Spring Security + JWT       |
| 接口文档 | Knife4j / SpringDoc OpenAPI |
| 对象存储 | MinIO / 云 OSS              |
| 构建工具 | Maven                       |
| 部署     | Docker / Docker Compose     |

## 7.3 前端架构

推荐前端技术栈：

| 类型            | 技术                     |
| --------------- | ------------------------ |
| 基础框架        | Vue 3                    |
| 开发语言        | TypeScript               |
| 构建工具        | Vite                     |
| 状态管理        | Pinia                    |
| 路由            | Vue Router               |
| UI 组件库       | Element Plus             |
| 请求库          | Axios                    |
| Markdown 编辑器 | MdEditorV3 / Markdown-it |
| 图表            | ECharts                  |
| 代码规范        | ESLint + Prettier        |
